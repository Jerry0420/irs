# IRS — 線上即時回饋系統

免登入的現場互動投票系統：觀眾掃 QR Code 進入手機投票頁，主持人在後台切換題目、
觀看即時數據；結果以「文字雲」與「圖片插針」動態呈現。溫暖人文木質風視覺。

## 快速開始（Docker）

```bash
docker-compose up -d --build
# 新版 Docker 亦可：docker compose up -d --build
```

| 頁面 | 網址 |
|------|------|
| 觀眾投票頁 | `http://<主機IP>:3000/` |
| 主持人後台 | `http://<主機IP>:3000/admin` |

現場觀眾手機須與主機同一區網（或主機有對外 IP）。後台頁面會自動產生觀眾頁 QR Code。

題目與上傳圖片存於 `./data/`（volume 掛載），容器重啟不遺失；
**票數保存在記憶體**，重啟容器等同全部重新投票。

## 功能

- **主持人後台**（免登入）：題目 CRUD 與排序（↑↓ 調整題號順序）、所有題目即時票數與縮小預覽、
  「切換題目」（觀眾端即時自動同步）、「清除數據／重新投票」。
- **題型**：
  - 文字題（n 個選項，n ≥ 2）→ 文字雲：得票越多字級即時平滑放大。
  - 四格圖片題 → 圖片插針：觀眾點圖投票，點擊處冒出大頭針，越熱門越密集。
- **觀眾端**：免登入、手機優先、大點擊區域；每題一票，投完鎖定顯示
  「投票成功，請等待下一題」，可**取消投票重新選擇**；主持人切題或清除數據後自動解鎖。
- **全螢幕結果投影**（後台內建）：後台按「🖥 全螢幕顯示目前題目」即以全螢幕呈現目前題目的
  文字雲／插針結果、題號、總票數與加入投票的 QR Code，下方附「上一題／下一題／清除數據」控制列，
  適合直接投放大螢幕。

## 開發模式（不經 Docker）

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 3000
```

### 跑測試

```bash
.venv/bin/python -m pytest tests/ -q
```

## 架構

- 後端：FastAPI + 原生 WebSocket（`/ws`），REST API 供後台 CRUD。
- 前端：無框架原生 HTML/CSS/JS，零建置；`ws-client.js` 負責斷線自動重連。
- 即時同步：投票、切題、清數據皆透過 WebSocket 廣播；投票廣播以 100ms 節流合併。
- 詳細設計見 [docs/ai-plan.md](docs/ai-plan.md)。

## 已知限制

- 每題一票採 localStorage + 連線層去重的**軟性限制**，無痕視窗可繞過。
- 後台以固定密碼保護（預設 `0000`，寫死於 `app/main.py` 的 `ADMIN_PASSWORD`），
  輸入一次後記在該瀏覽器內；屬輕量保護，仍不建議將 3000 埠暴露到不信任的網路。
- 單一活動場次，無多房間概念。
